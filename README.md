# skin_lesion_processor
皮肤病灶轮廓处理系统——支持图像轮廓提取、NURBS曲线拟合、自动排样、激光切割模拟与云端数据管理的一站式桌面应用。

---

## ✨ 主要功能

- **图像轮廓提取**  
  基于 OpenCV 自动检测图像中的病灶轮廓，支持闭运算等预处理，过滤小面积噪点。

- **NURBS / 贝塞尔曲线拟合**  
  使用 NURBS 或三次贝塞尔曲线对轮廓进行平滑拟合，可动态调整控制点数量，实时预览拟合效果。

- **多图像管理与标号映射**  
  每张图像对应唯一标号，记录来源（本地/云端）、创建时间、删除状态，支持标号映射表查看与导出。

- **自动排样（极坐标布局）**  
  基于 Shapely 的极坐标排样算法，将轮廓紧凑排列在直径 10cm 的圆形画布中，支持多页翻页。

- **尺寸标定**  
  通过检测原始 overlay 图像中的白色小球（直径 10mm）自动计算像素比例尺，赋予轮廓真实物理尺寸。

- **DXF / JSON / 图像导出**  
  支持将轮廓导出为 DXF（用于激光切割）、JSON（用于数据交换）或直接保存画布截图（PNG/JPEG/BMP）。

- **激光切割模拟**  
  内置激光控制器（支持模拟模式与硬件模式），可加载 DXF 文件模拟切割过程，实时显示进度与轨迹，生成模拟报告。

- **云端数据集成**  
  通过微信云开发 API 获取患者列表，批量下载 mask 与 overlay 文件，自动构建文件映射，支持增量添加。

- **交互式画布**  
  支持轮廓的拖拽、缩放、非均匀拉伸，显示包围盒与控制点，可切换显示原始轮廓/NURBS曲线/标号。

## 📦 技术栈

- Python 3.8+
- PyQt5 – GUI 框架
- OpenCV – 图像处理与轮廓提取
- NumPy, SciPy – 数值计算
- shapely – 几何排样
- geomdl – NURBS 曲线拟合
- ezdxf – DXF 文件导出
- requests – 云端 API 调用

### 安装依赖

主要依赖库：PyQt5、opencv-python、numpy、shapely、geomdl、ezdxf、requests、scipy


## 基本操作流程
加载图像：点击“加载多个图像文件”选择 mask 图像（通常为黑白二值图）。

处理图像：点击“处理所有图像”提取轮廓并自动标定尺寸（需配套 overlay 图像）。

调整轮廓：选中轮廓后可在右侧面板修改尺寸、控制点数，画布支持拖拽与缩放。

自动排样：点击“自动排列轮廓”将当前所有轮廓极坐标排样，自动分页。

导出数据：支持导出 DXF、JSON、图像等格式。

激光切割：切换到“激光控制”面板，选择模拟模式，点击“执行激光切割”模拟切割过程。

## ☁️ 云端数据使用
在右侧“云端数据”停靠窗口点击“刷新患者列表”获取云端患者。

选择一名或多名患者，点击“下载并处理选中患者”将完全替换当前项目；

点击“添加选中患者数据”则将数据增量添加到当前项目。

下载完成后程序会自动标定轮廓尺寸（通过 overlay 图像中的白色小球）。


## 🛠️ 常见问题
Q: 为什么自动标定没有成功？
A: 请确保原始 overlay 图像中包含完整的白色小球（直径 10mm），且文件命名符合 *_overlay.png 格式，与 mask 文件名匹配。

Q: 云端下载后程序闪退？
A: 该问题已通过优化线程生命周期管理和内存回收解决。如仍出现，请检查网络连接或文件完整性。

Q: 控制点滑块不更新？
A: 选中不同轮廓时，控制点滑块会自动同步当前轮廓的控制点数。如有异常请重启程序。
